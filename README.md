## About

This repository operationalizes a three-phase information systems (IS) proposal for **Dataveil**, a fictitious mid-sized wealth management firm headquartered in Albany, NY, serving ~12,000 high-net-worth clients with ~$4.2B in assets under management. The project demonstrates two AI-enabled IS capabilities on representative synthetic data:

1. **Real-time fraud and anomaly detection** — Isolation Forest (unsupervised) + Logistic Regression (supervised) ensemble on 50,000 transactions with a 1.2% fraud rate.
2. **Client segmentation for personalization** — K-means clustering on 12,000 clients, yielding four interpretable advisor-facing personas.

All data is synthetic and engineered to mirror publicly documented financial patterns. No real client information is used.


### Reproduce the analysis

```bash
# Regenerates both datasets from a fixed seed and reruns both models end to end
python src/reproduce_analysis.py

# Regenerates all seven figures from the data
python src/make_figures.py
```

Outputs are written to the working directory. The random seed is fixed at `42` for full reproducibility.

## Data

Both datasets are synthetic, generated by `reproduce_analysis.py`. See **Appendix C of the Phase 2 report** for the full generative model.

### `dataveil_transactions.csv` — 50,000 rows

| Column | Type | Description |
|---|---|---|
| `amount` | float | Transaction amount (USD) |
| `hour` | int | Hour of day, 0–23 |
| `txn_velocity_1h` | int | Transactions in the prior hour |
| `acct_age_days` | int | Account age in days |
| `amount_to_avg_ratio` | float | Transaction ÷ account's historical average |
| `cross_border` | int | Cross-border flag (0/1) |
| `is_fraud` | int | Ground-truth label (0/1) |

**Class balance:** 49,400 legitimate (98.8%) · 600 fraudulent (1.2%)

### `dataveil_clients.csv` — 12,000 rows

| Column | Type | Description |
|---|---|---|
| `age` | int | Client age, 22–92 |
| `aum` | float | Assets under management (USD) |
| `risk_tolerance` | float | Risk score, 1 (most conservative) to 10 (most aggressive) |
| `tenure_yrs` | int | Years as a client, 0–24 |

| `products_held` | int | Number of products, 1–7 |

`dataveil_clients_segmented.csv` adds `segment` (cluster id) and `persona` (descriptive name) columns.

## Methodology

The analysis follows **CRISP-DM** — Cross-Industry Standard Process for Data Mining — moving from business understanding through data preparation to modeling and evaluation.

### Analysis A — Fraud Detection

- 70/30 stratified train/test split, preserving the 1.2% fraud rate
- StandardScaler on all six features
- **Isolation Forest** (200 trees, contamination = 0.012)
- **Logistic Regression** (`class_weight='balanced'`)
- Evaluation on the 15,000-row held-out test set using **precision, recall, F1, and AUC** — not raw accuracy, which is misleading under severe class imbalance

### Analysis B — Client Segmentation

- StandardScaler on all six features
- Cluster count selected via **elbow method + silhouette analysis** across k = 2…8
- **K-means** at k = 4 (best balance of statistical fit and interpretability)
- Personas assigned by the most distinctive trait per cluster
- 2D visualization via PCA (~45% variance retained)

## Key Results

### Fraud detection (held-out test set, n = 15,000)

| Model | Precision | Recall | F1 | AUC |
|---|---|---|---|---|
| Isolation Forest | 0.910 | 0.894 | 0.902 | 0.999 |
| Logistic Regression | 0.746 | 0.994 | 0.852 | 1.000 |

The two models trade off differently: logistic regression catches nearly all fraud but flags more legitimate transactions; the Isolation Forest is more balanced and requires no labeled fraud history. **The recommended deployment is an ensemble** that combines both, with the alert threshold governed as a business decision rather than an algorithmic one.

### Client segmentation (k = 4)

| Segment | n | Mean age | Mean AUM | Risk (1–10) | Tenure (yr) |
|---|---|---|---|---|---|
| High-Net-Worth Core | 789 | 61 | $1,223,925 | 5.2 | 13 |
| Established Loyal Clients | 3,484 | 59 | $263,151 | 4.2 | 18 |
| Conservative Wealth Preservers | 3,542 | 63 | $265,431 | 3.8 | 6 |
| Emerging Growth Seekers | 4,185 | 46 | $241,440 | 8.1 | 11 |

The **High-Net-Worth Core** is the smallest segment but concentrates roughly five times the mean AUM of other segments — concentrating both revenue and retention risk.

## Reproducibility Notes

- All randomness is seeded at `SEED = 42` in `reproduce_analysis.py`.
- Cluster numbering from K-means is not deterministic across scikit-learn versions, but persona assignment (by most distinctive trait) is stable; segment sizes and profiles should match the reported numbers within rounding.
- Figures are regenerated from the same data — re-running `make_figures.py` against a freshly generated dataset reproduces the published charts.

## Limitations

This is a course deliverable on synthetic data, not a production system. Key limitations documented in the milestone paper:

- Near-perfect detection metrics reflect engineered separability; real fraud is noisier and adversarial.
- The NLP compliance pillar from Phase 1 is treated qualitatively only.
- No fairness audit, no cost-sensitive evaluation, no adversarial robustness testing.
- K-means results are sensitive to feature scaling and cluster count.

See **Limitations** and **Suggestions for Subsequent Analysis** in the milestone paper for the full discussion.
